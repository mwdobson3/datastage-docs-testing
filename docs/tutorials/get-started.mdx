## Prerequisites

Before starting this tutorial series, ensure you have:

- **Kubernetes Cluster** - Access to EKS, AKS, GKE, or OpenShift
- **Admin Access** - Ability to install software on your local machine
- **IBM Cloud Pak for Data** - An active instance (trial or production)
- **Basic Kubernetes Knowledge** - Familiarity with pods, deployments, and services

:::tip Estimated Time
The complete tutorial series takes approximately **40 minutes** to complete.
:::

## What is DataStage Anywhere?

DataStage Anywhere allows you to deploy IBM DataStage Remote Engines on any Kubernetes cluster, enabling you to:

- **Run DataStage jobs closer to your data** - Reduce latency and data transfer costs
- **Scale independently** - Add compute resources without affecting Cloud Pak for Data
- **Support hybrid architectures** - Process data in on-premises, cloud, or edge environments
- **Maintain centralized governance** - Manage all engines from Cloud Pak for Data
