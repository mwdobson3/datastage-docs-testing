## Prerequisites

Before starting this tutorial, ensure you have:

- **AWS Account** - With permissions to create/access EKS clusters
- **IBM Cloud Account** - With access to Cloud Pak for Data
- **kubectl** - Kubernetes command-line tool installed
- **AWS CLI** - For EKS cluster access configuration
- **IBM Cloud API Key** - For authentication
- **Basic Kubernetes Knowledge** - Understanding of pods, deployments, namespaces

:::tip For IBMers
You can use [IBM TechZone](https://techzone.ibm.com/) to provision a pre-configured EKS cluster for testing.
:::

:::tip Estimated Time
The complete tutorial series takes approximately **60 minutes** to complete.
:::

## Key Features

- **Managed Kubernetes**: AWS handles cluster management and scaling
- **High Availability**: Multiple compute pods for workload distribution
- **Persistent Storage**: EFS integration for data persistence
- **Auto-scaling**: Kubernetes-native scaling capabilities
- **Production Ready**: Enterprise-grade deployment architecture

## Comparison: EKS vs Docker

| Feature | AWS EKS (Kubernetes) | AWS EC2 (Docker) |
|---------|---------------------|------------------|
| **Complexity** | Higher | Lower |
| **Scalability** | Automatic | Manual |
| **High Availability** | Built-in | Requires setup |
| **Management** | AWS managed | Self-managed |
| **Cost** | Higher (cluster costs) | Lower (single VM) |
| **Best For** | Production workloads | Development/Testing |
