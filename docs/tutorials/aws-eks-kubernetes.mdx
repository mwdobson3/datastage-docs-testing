# Deploy DataStage Remote Engine on AWS EKS

Learn how to deploy a DataStage Remote Engine on Amazon Elastic Kubernetes Service (EKS) to run data integration workloads close to your data sources while maintaining centralized control through IBM Cloud Pak for Data.

## Why Deploy on Kubernetes?

Kubernetes provides enterprise-grade orchestration for containerized DataStage engines with:

- **High Availability**: Automatic pod rescheduling and self-healing
- **Scalability**: Dynamic resource allocation based on workload demands
- **Resource Efficiency**: Optimal utilization of compute resources
- **Enterprise Security**: Network policies, RBAC, and secrets management
- **Multi-tenancy**: Run multiple engines in isolated namespaces

### Key Components

**Design-time on IBM Cloud**
- Manage projects and user access
- Design DataStage jobs using the web-based canvas
- Import existing DataStage assets
- Administrative tools and monitoring

**DataStage Runtime Execution**
- Initiates API calls to remote engine
- Pulls job metadata and orchestrates execution
- Firewall allows only outbound connections for security

**Remote Engine (On-premises or Client VPC)**
- **PX Runtime (Conductor)**: Orchestrates parallel job execution
- **PX Compute**: Parallel processing containers for data transformation
- **Persistent Volume**: Stores engine configuration and temporary data
- Deployed on Docker or Kubernetes infrastructure

### Benefits

✅ **Retain full data privacy** - Data never leaves your environment  
✅ **Reduce egress/ingress fines** - Minimize unnecessary data movement  
✅ **Improved performance** - Process data where it resides  
✅ **Retain data sovereignty** - Comply with data locality regulations

## What You'll Learn

This tutorial walks you through:

1. **Prerequisites and Setup** - Required tools and access
2. **Provision EKS Cluster** - Create Kubernetes cluster on AWS
3. **Configure kubectl Access** - Connect to your EKS cluster
4. **Prepare Configuration** - Set up engine parameters
5. **Deploy Remote Engine** - Install DataStage on EKS
6. **Monitor and Verify** - Ensure successful deployment

## Prerequisites

Before starting, ensure you have:

- AWS account with EKS permissions
- IBM Cloud Pak for Data instance (SaaS or on-premises)
- Basic knowledge of Kubernetes and AWS
- Command-line tools: `kubectl`, `aws-cli`

## Time Estimate

⏱️ **Total Time**: 45-60 minutes

- Cluster provisioning: 15-20 minutes
- Configuration: 10-15 minutes  
- Deployment: 15-20 minutes
- Verification: 5-10 minutes
